1) Introduction

Programming LANguage 9 (PLAN9) is a simple programming language intended to be used in teaching beginner programmers the fundamental concepts needed to advance to more complex programming languages. The Approach taken by PLAN9 is to use simple instructions in plain English to teach fundamental concepts. We believe that this approach will allow students to more easily understand the fundamental programming concepts by allowing them to easily make the connection between the simple instructions to the more advanced instructions found in C++ & Java.

PLAN9 is made simple and easy to use by the small number of Instructions it supports. PLAN9 supports only nine different instructions. With these 9 simple instructions however, students will be able to create powerful applications in a short period of time while learning the basic fundamental concepts of programming. This approach allows the students to comprehend the fundamentals without cluttering there minds with lots of technical jargon.

PLAN9 was created in the Fall of 2004 by Professor John Stamey & Bryan Saunders at Coastal Carolina University. It was developed as new way to teach programming in the CSCI 130 course.

 

2) The Development Environment

The Online Development Environment for PLAN9 is a Java Applet written by Bryan Saunders. The Environment will allow user to check there code for errors & visually see what the code does. The environment has a built in debugger that will check the students code for any errors and inform them of any errors that it might find. The Most recent build can be launched from the link below.

2.1) Reading the Output

The output created by the Enviroment is a grid. Along the Top & Left side are the Column and Row Numbers. Where they Intersect creates a grid location for the UFO. When the UFO moves from one Grid Location to the next, it leaves a * in its trail. If the UFO is hidden however, it will not leave anything in its trail and will only continue to leave a * when it is shown again. You may see several other characters within the grid as well. The Letters N, S, E, W All mark the UFOs Starting and Ending locations and stand for his Direction at that time. So an E located at grid location 2,3 means the UFO started on Row 2 in Column 3 Facing East or it could also mean that he Ended on Row 2 in Column 3 Facing East.

The Printout at the bottom of the Grid, displays the pertinent information about the robots Final Location.

 

3) PLAN9 Instruction Set

The Code for PLAN9 is very simple and uses "Plain English" terms. This makes it much easier for new programmers to learn how to program. PLAN9 will support 7 Primary Instructions

3.1) DIR

Proper Use: DIR(X);
Possible Value for X: N, S, E, W

The DIR Command is used to instruct the UFO to turn in place and face a new Direction. The UFO will turn to face the new direction issued to is by the command. N is North (Up), S is South (Down), E is East (Right), & W is West (Left). So the command DIR(N) would force your UFO to face North.

Supported in: v0.3 & Up

3.2) MOVE1

Proper Use: MOVE1;

The MOVE1 Command is used to instruct the UFO to move 1 grid location in the direction is is currently facing.

Supported in: v0.2 & Up

3.3) INIT

Proper Use: INIT(X);
Possible Value for X: -127 to 128

The INIT Command is used to turn on, or Initialize your counter. The X value is the value that you want it to start at. So the Command INIT(6) will cause your counter to start at 6. You MUST Initialize your counter before you can Change or Set its value.

Supported in: v0.3 & Up

3.4) SET

Proper Use: SET(X);
Possible Value for X: -127 to 128

The SET Command will reset your counter the value of X. So SET(3) will reset your counter to the value of 3. You MUST Initialize your counter before you can Change or Set its value.

Supported in: v0.3 & Up

3.5) CHANGE

Proper Use: CHANGE(X);
Possible Value for X: -127 to 128

The CHANGE command will add the supplied value to your counter. If you want to add 3 to your counter you would use CHANGE(3). Likewise, if you want to subtract 5 you would issue the command CHANGE(-5). You MUST Initialize your counter before you can Change or Set its value.

Supported in: v0.3 & Up

3.6) HIDE

Proper Use: HIDE;

The HIDE command will cause the UFO to "cloak" himself. When the UFO is in this state, it will not leave a * or any other marking in its path. It will only leave a marking in its path if it stops and is still hidden, and in this case, the marking will be the letter of the direction he was last facing. Can not run HIDE if robot is already Hidden.

Supported in: v0.4 & Up

3.7) SHOW

Proper Use: SHOW;

The SHOW command causes the UFO to "un-cloack" himself once again becoming visible. When the UFO is un-cloaked it will leave a * in its trail. can not run SHOW unless you have already run HIDE.

Supported in: v0.4 & Up

3.8) LOOP / ENDLOOP

Proper Use: LOOP(X); / ENDLOOP;
 Possible Value for X: -127 to 128

The LOOP command allows you to repeat code. You simply replace the value of X with the number of times that you want the loop to run. so LOOP(2) will cause whatever is in the loop code block to run 2 times. You end the loop code block with the instruction ENDLOOP. An Example of a loop would look like:

LOOP(2);
MOVE1;
ENDLOOP;

This code would make the UFO execute the MOVE1 command 2 times.

Supported in: v0.5 & Up

3.9) VIEW

Proper Use: VIEW;

The VIEW command informs the compiler that you want to see the final value of your counter at the end of the programs run. This command can be issued anywhere in the course of the programs execution however the best use of this command is to issue it at the very end or beginning of the program

Supported in: v0.9 & Up

 

4) Troubleshooting

If you are having problems with the Applet, please read the sections below. If you still can not get the applet to run, then you may email Bryan Saunders for further help. Please allow 1-3 days for a response.

4.1) Error Codes

If you have an Error in your code, you will get one of several messages displayed in the top box of the applet. The Messages are displayed in the format "ERROR::LINE". Line is the Line number in your code that the error is on. Error is the problem in the code. Possible Error messages are:

INVALID INSTRUCTION The Instruction is not a valid Instruction
VAR NOT INITIALIZED You Tried to Change/Set the Var but never Initialized it
VAR ALREADY INITIALIZED The Var is already Initialized. but you tried to Initialize it again
INVALID INT You tried to Change/Set to a Value that is invalid
INVALID DIR You Entered a Direction other than N, S, E, or W
INVALID START INFO Starting Information Provided is Invalid
MOVES OFF COURSE Issued MOVE1 Command takes UFO out of grid
NOT HIDDEN The UFO is not yet hidden, can not run SHOW yet
ALREADY HIDDEN The UFO is already hidden, can not run HIDE again
INVALID SYNTAX ';' All Lines must end with a Semi-Colon ';'
ALREADY IN LOOP Tried to Start a New Loop within a Loop
LOOP NOT STARTED Can Not ENDLOOP if No Loop Started
ERROR SAVING CODE There was an Error when writing to code file. Contact Bryan Saunders.

4.2) Applet Wont Run

If your Applet will not run, it is most likely being cause by one of two reason. The first reason is most likely that you did not Approve the Security Signature. You MUST Allow the applet to run by approving the Security Warning. To Restart the applet to approve the Security Warning, simply refresh the page with the Applet on it. If Refreshing does not work, close the browser completely, then re-open the browser and re-launch the applet.

Another likely cause is the lack of the Sun Java 2 Virtual Machine. This can be downloaded from the Sun Microsystems Website. If you do not have the latest Sun Java Virtual Machine, and can not find it on there website, the link is provided for you below. The Latest JVM will also be automatically installed for you the first time you try to run the applet and you do not have it.

Get The Java 2 Virtual Machine 1.4.2_05

 

5) Revisions

Here is a Brief History of the Previous Versions of PLAN9

5.1) Version 0.1

Version 0.1 was never actually coded up. This version was the general idea for the language that was presented by Professor John Stamey.

5.2) Version 0.2

Version 0.2 was online for a short time. It supported 5 primitive instructions. Those instructions were NORTH, SOUTH, EAST, WEST, & MOVE1. The Directions were all condensed into the DIR instruction for Version 0.3. Only the MOVE1 Command was carried over.

5.3) Version 0.3

Version 0.3 Implemented new Error & Syntax Checking Routines.

5.4) Version 0.3.5

Version 0.3.5 Expanded v0.3 by better implementing the new Routines and also properly displayed the Final Printout.

5.5) Version 0.4

Version 0.4 Implemented two new instructions, HIDE and SHOW. These are used to hide & show the UFO respectively. We also changed the Letter code for the Robots Final Location to display its Direction instead of an F.

5.6) Version 0.5

Version 0.5 Implemented tow new instructions. They are LOOP and ENDLOOP. This gives PLAN9 looping functionality.

5.7) Version 0.6

Version 0.6 Implements several code improvements. These improvements primarily streamline the code and make it more maintainable. Other than faster performance from the Applet, the user will not notice any differences.

5.8) Version 0.7

Working off of the new Streamlined code, a Small text box was added to the right side of the Applet for users to enter there code directly into the system. Users can still open a text file containing code also.

5.9) Version 0.8

Implemented the Ability for users to save there code after they have made changes to it in the text box or if they started by typing there code directly into the text box.

5.10) Version 0.9

Fixed Error with Using CHANGE / SET in Loops. also implemented the ninth instruction, VIEW. Debugging Console Enabled.

5.11) Version 0.9.5

Fixed Error Checking Method for HIDE/SHOW. Was Giving False Error Messages. Fixed Bug where LOOP commands where executing more than told to due to the previous loop code not being properly cleared.

5.12) Version 1.0

Finalized Code, Fixed a few Remaining Bugs.
